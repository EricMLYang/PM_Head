# PM 總管 — 設計決策記錄

---

## 2026-03-06 — 初始鍛造決策

### 決策 1：角色定位為 Expert Repo
- **選項**：Expert Repo vs Project Repo
- **選擇**：Expert — PM 總管的知識跨多個專案複用
- **理由**：PM 總管管理 MI_PM、VMS_PM、SmartSignagePM 三條產品線，其策略思維和彙整方法可跨專案複用

### 決策 2：不複製子 PM 的 Discovery 流程技能
- **選項**：包含 Inbox→OPP→Backlog 流程技能 vs 只做跨專案管理技能
- **選擇**：只做跨專案管理技能，Discovery 流程留給子 PM
- **理由**：PM 總管的視角是俯瞰而非執行，越級干預是核心反模式

### 決策 3：加入 util-pptx 技能
- **選項**：簡報能力由外部處理 vs 內建 util-pptx
- **選擇**：內建 — PM 總管常需對上層做策略簡報
- **理由**：定期報告和策略分析經常需要簡報形式交付

### 決策 4：以 PM 原型微調（路徑 b）
- **選項**：(a) 直接複製 (b) 原型微調 (c) 從空白開始
- **選擇**：(b) 微調 — PM 底層思維不變，但視角大幅調整
- **理由**：保留 PM 核心概念（需求、優先順序、利害關係人），新增跨專案管理視角
